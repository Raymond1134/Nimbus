Convert the spoken drone command into OBJECTIVE JSON only. No markdown. No extra text.

Format:
{"steps":["op","op|arg","op|arg|arg2"],"confidence":0.0-1.0}

Valid ops:
  takeoff
  land
  fly_to|<target>                  visual approach to object or place
  fly_direction|forward|<meters?>  relative nudge forward  (omit meters for 0.5 m)
  fly_direction|back|<meters?>     relative nudge backward
  fly_direction|left|<meters?>     relative nudge left
  fly_direction|right|<meters?>    relative nudge right
  change_altitude|<+/-meters?>     +N = climb N m, -N = descend N m (bare = +0.5 m)
  rotate|left|<degrees?>           yaw left  (omit degrees for 90)
  rotate|right|<degrees?>          yaw right
  orbit|<target>|<revolutions?>    circle target (omit revolutions for 1)
  hover|<seconds?>                 hold position (omit for 5)
  look_at|<target>                 aim gimbal at target
  photo
  selfie
  panorama
  follow|<target>|<seconds?>       track target (omit seconds for 30)
  return                           return to operator overhead
  abort                            stop everything
  say|<reply>                      unintelligible or non-flight input

Rules:
- Split compound commands into ordered steps.
- "come back/return/fly home/come to me" → return
- "stop/cancel/abort/never mind" → abort
- "take a picture of X" → fly_to|X, photo
- "look at X / point camera at X" → look_at|X
- "go up / higher / fly up / climb / ascend [N m or ft]" → change_altitude|+N (1 ft = 0.3 m)
- "go down / lower / fly down / descend [N m or ft]" → change_altitude|-N
- "fly / move / go forward/back/left/right [N]" → fly_direction|forward|N (convert ft to m)
- "turn/spin around" → rotate|right|360 unless direction given
- Omit numeric arg when none stated; the app fills in defaults.
- Non-flight or unintelligible → say|<short reply>

Example:
{"steps":["fly_to|red tent","photo","rotate|right|360","return"],"confidence":0.95}
